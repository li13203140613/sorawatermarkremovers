# 博客 AI 辅助创作指南

## 🚀 快速开始

使用新的 AI 增强版博客创建工具，您只需要提供：
1. **文章标题**
2. **文章描述**
3. **核心要点**（3-5 个）

工具会自动生成带有 AI 提示标记的文章框架，然后您可以让 AI 助手（如 Claude Code）填充内容。

---

## 📝 完整工作流程

### 第 1 步：创建文章框架

```bash
npm run blog
```

按提示输入信息：

```
📝 文章标题（中文）: 如何使用 AI 工具提升视频制作效率
📝 文章标题（英文）: How to Improve Video Production Efficiency with AI Tools
📄 文章描述（中文）: 介绍最新的 AI 视频制作工具及其实用技巧
📄 文章描述（英文）: Introduce the latest AI video production tools and practical tips
👤 作者名: [直接回车使用默认]
🏷️  标签: AI工具,视频制作,效率提升
✨ 添加文章核心要点？(Y/n): Y
📌 请输入文章的核心要点（每行一个）:
  - AI 视频编辑工具推荐
  - 自动化字幕生成技巧
  - AI 视频增强功能
  - 批量处理视频的方法
  - 成本效益分析
  - [输入空行结束]
🌐 创建双语版本？(Y/n): Y
```

### 第 2 步：查看生成的文章框架

### 第 3 步：使用 AI 填充内容

#### 方法 A：使用 Claude Code（推荐）

1. 在 VS Code 中打开生成的 MDX 文件
2. 选中第一个【AI 提示】标记和相关内容
3. 对 Claude Code 说：

```
请根据这个 AI 提示，生成 500-800 字的详细内容。
要求：
- 内容充实，有实例
- 包含代码示例或配置示例（如适用）
- 使用 Markdown 格式
- 包含适当的列表、表格或引用
```

4. 重复步骤 2-3，填充所有章节

#### 方法 B：使用 ChatGPT/Claude

1. 复制整个 MDX 文件内容
2. 发送给 AI，并说：

```
这是一篇博客文章框架，请帮我填充所有【AI 提示】标记的内容。
要求：
- 每个章节 500-800 字
- 保留 frontmatter 和 Markdown 结构
- 内容专业、实用
- 包含具体例子
```

3. AI 会返回完整的文章
4. 复制粘贴回 MDX 文件

#### 方法 C：逐个章节填充

1. 复制单个章节的 AI 提示
2. 发送给 AI：

```
标题：如何使用 AI 工具提升视频制作效率
章节：AI 视频编辑工具推荐

请为这个章节生成 500-800 字的详细内容，包括：
- 工具推荐（3-5 个）
- 每个工具的特点
- 使用场景
- 优缺点对比
```

3. 依次填充每个章节

### 第 4 步：审核和优化

1. **检查内容质量**
   - 删除所有【AI 提示】标记
   - 确保内容连贯
   - 检查专业术语准确性
   - 验证代码示例可运行

2. **添加视觉元素**
   - 插入相关图片
   - 添加表格对比
   - 使用引用块突出重点

3. **SEO 优化**
   - 确认标题包含关键词
   - 描述简洁明了
   - 标签准确相关

4. **预览文章**
   ```
   访问: http://localhost:3000/blog/your-article-slug
   ```

## 💡 AI 提示词最佳实践

### 开头引言提示词

```
请根据以下信息生成博客文章的开头段落（150-200字）：

标题：[文章标题]
描述：[文章描述]
目标读者：[RemoveWM 用户/视频创作者/开发者]

要求：
- 引起读者兴趣
- 概括文章主要内容
- 提出问题或痛点
- 自然过渡到正文
```

### 章节内容提示词

```
请为以下章节生成详细内容（500-800字）：

章节标题：[章节标题]
文章主题：[文章标题]

要求：
- 结构清晰，使用子标题
- 包含3-5个具体要点
- 提供实际例子或代码
- 使用列表或表格（如适用）
- 专业但易懂
```

### 总结章节提示词

```
请为文章生成总结段落（200-300字）：

文章标题：[标题]
主要章节：[列出所有章节标题]

要求：
- 总结核心要点（3-5条）
- 给出行动建议
- 鼓励读者尝试
- 引导到相关资源
```

## 🔧 高级技巧

### 技巧 1：批量生成多篇文章

创建多个框架后，使用 AI 批量处理：

```bash
# 创建第一篇
npm run blog

# 创建第二篇
npm run blog

# 创建第三篇
npm run blog

# 然后让 AI 逐个填充内容
```

### 技巧 2：保存常用提示词模板

在 `scripts/prompts/` 创建提示词模板文件：

```markdown
<!-- intro-prompt.md -->
请根据以下信息生成博客开头段落...
[具体要求]

<!-- section-prompt.md -->
请为以下章节生成内容...
[具体要求]
```

### 技巧 3：使用变量替换

修改脚本支持变量替换，让提示词更具体：

```javascript
// 将来可以扩展
const prompt = `请为"${sectionTitle}"章节生成内容...`;
```

### 技巧 4：结合现有文章

告诉 AI 参考您已有的优质文章风格：

```
请参考这篇文章的写作风格：[链接或内容]
然后为新文章生成内容...
```

## ⚠️ 注意事项

1. **不要完全依赖 AI**
   - AI 生成的内容需要人工审核
   - 检查事实准确性
   - 确保符合品牌调性

2. **保护敏感信息**
   - 不要在 AI 提示中包含密钥
   - 避免泄露用户数据
   - 遵守隐私政策

3. **版权和原创性**
   - AI 生成的内容可能不完全原创
   - 重要文章建议人工重写
   - 添加独特见解和经验

4. **保持一致性**
   - 统一术语使用
   - 保持文章风格一致
   - 维护品牌形象
